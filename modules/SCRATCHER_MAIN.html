МОДУЛЬ SCRATCHER

Анализ требований и существующих решений

Модуль основного продукта представляет продвинутый парсер и поисковик по российским маркетплейсам с аналитикой собранных данных. Задача – помочь как начинающим предпринимателям найти перспективные ниши, так и крупным селлерам получить аналитику конкурентов. Ключевые запросы: сбор данных о продажах, обороте и остатках товара у конкурентов на Wildberries, Ozon, Яндекс.Маркете, а также анализ аудитории по отзывам (определение целевых групп). Идея – получить всю маркетинговую информацию (объемы продаж, целевая аудитория, тренды) на основе собранных данных.

Сейчас на рынке уже существуют подобные сервисы: например, расширение «Маяк» показывает продажи и обороты товаров на WB и Ozon, а расширение Seerfar предлагает продавцам аналитику продаж, трендов и конкурентов. Эти продукты являются бесплатными или условно-бесплатными инструментами для мониторинга ниши. Это говорит о наличии спроса, но нам нужно выделиться «фишками»: интеграция в Telegram-ботов, использование AI для анализа отзывов, персональная статистика.

Сильные стороны текущей концепции

  • Единая система на базе Telegram-ботов и n8n
  • Возможность автоматизации процессов
  • Наличие AI-компонентов (OlLama, генерация контента)
  • Функционал ботов уже покрывает администрирование и лидогенерацию

Слабые стороны и риски

  • Анти-бот защиты и изменения API маркетплейсов
  • Конкуренция с готовыми решениями («Маяк», Seerfar)
  • Требования к хранению и обработке больших объёмов данных
  • Сложность NLP/ML для анализа отзывов
  • Правовые ограничения на парсинг и нагрузку на сайты

Функциональные требования и варианты развития

Маркетплейсы и приоритеты

Приоритетная очередь: Ozon → Wildberries → Яндекс.Маркет. Сначала Ozon и WB (много открытой информации), затем подключить ЯМ (API партнёра или сложный парсинг).

Режим сбора данных

Реального времени не требуется: сбор по запросу (клик пользователя) или cron-триггер. Это упрощает архитектуру и снижает нагрузку.

Тип выводимой информации

  • Telegram-бот (с Web App) — быстрота запросов
  • Графический интерфейс («оболочка» на Docker/OlLama/n8n) — детальный анализ. Данные сохраняются в базу или Excel.

База и форматы

Начальный этап — Excel/CSV. При росте — перейти на СУБД:

  • SQL (PostgreSQL/MySQL) — структурированные данные
  • NoSQL (MongoDB) — гибкость схемы для отзывов

Отчетность и визуализация

Генерация графиков (Matplotlib/Plotly) и отправка в Telegram или Web App. Telegram Mini Apps для HTML5-интерактивности.

Дополнительные фичи

  • Анализ остатков конкурентов
  • Тренды и история продаж (динамика)
  • Анализ ЦА по отзывам (LLM, ключевые слова)
  • Мониторинг изменений цен и остатков у конкурентов

Технологический стек и архитектура

Сбор данных (парсинг)

Python/Node.js, парсинг HTML/API. Для WB — внутреннее API через XHR. Теги: requests, BeautifulSoup, Scrapy, Playwright/Selenium. Прокси и смена User-Agent.

Хранение данных

SQL (PostgreSQL/MySQL) для таблиц, NoSQL (MongoDB) для отзывов. Индексация, бэкапы.

Обработка и аналитика данных

Python: pandas, NumPy, scikit-learn, NLTK/spacy, transformers. LLM через OlLama для анализа отзывов и портретов аудитории.

Интеграция и автоматизация (n8n)

HTTP Request, узлы БД, Cron. Webhook из Telegram запускает воркфлоу. Интеграция с Google Sheets.

Telegram-бот

Библиотеки: python-telegram-bot, aiogram. Команды “анализ товара X”, графики, Web Apps.

Веб/графический интерфейс

Flask/Django или PyQt/Qt. Чтение данных из той же БД.

Развертывание (Docker)

Контейнеры: парсер, бот, веб-интерфейс, OlLama. Оркестратор — Docker.
